# autoimprove Phase 1 MVP — Implementation Design

**Date:** 2026-03-25
**Status:** Approved
**Builds on:** DESIGN.md, TELEMETRY.md

## Decisions Made

These deviate from or refine the original DESIGN.md based on brainstorming review:

1. **Claude Code plugin, not Python SDK app.** TELEMETRY.md is reference material. The orchestrator is a skill prompt, the experimenter is an agent prompt. Claude is the runtime.
2. **Set logic scoring, no weighted composite.** Keep rule: no metric regresses beyond tolerance AND at least one improves beyond significance. No weights. Per-metric tolerances are the control knob. Epoch drift check catches compound regression independently. (DESIGN.md's per-tier significance scaling — 1%/2%/3% — is also dropped; per-metric thresholds are sufficient.)
3. **Single `evaluate.sh` script as the prepare.py equivalent.** Deterministic evaluation code using `jq` for JSON manipulation and arithmetic. Claude orchestrates, evaluate.sh evaluates. JSON in, JSON out. One seam. Requires `jq` as a dependency.
4. **Agent personas deferred.** Add when stagnation data justifies it. V1 relies on themes + cooldowns + stagnation detection.
5. **Cost budget deferred.** V1 uses experiment count only (`max_experiments_per_session`). Token-to-dollar tracking requires model pricing lookups not available in-session. Add in post-v1.
6. **Worktree merge is fast-forward only.** Worktrees are created from current HEAD at each loop iteration. Merges rebase the experimenter's commit(s) onto main and fast-forward. Rebase failure = discard.

## Component Architecture

```
.claude-plugin/
  plugin.json                    # plugin manifest
  skills/
    orchestrator.md              # main loop — reads config, manages state, spawns experiments
    init.md                      # scaffold autoimprove.yaml for a new project
    report.md                    # morning report
  agents/
    experimenter.md              # runs in worktree, blind to scoring
  commands/
    run.md                       # /autoimprove run
    report.md                    # /autoimprove report
    init.md                      # /autoimprove init

scripts/
  evaluate.sh                   # runs gates, benchmarks, extracts metrics, applies set logic
                                # requires: jq, bash 4+

experiments/                    # created at runtime
  experiments.tsv               # append-only log
  epoch-baseline.json           # frozen at session start
  rolling-baseline.json         # updated after each KEEP
  state.json                    # trust tier, cooldowns, stagnation counters
  <id>/context.json             # per-experiment reproducibility record

test-project/                   # integration test fixture
  (minimal project with tests, benchmarks, known deficiencies)

docs/
  (user-facing documentation for all features)
```

**Key boundary**: Claude orchestrates (loop, state, agents). `evaluate.sh` evaluates (gates, benchmarks, scoring). JSON interface. One seam.

## evaluate.sh Contract

### Input

```bash
evaluate.sh <evaluate-config.json> <baseline.json>
```

`evaluate-config.json` (generated by orchestrator from autoimprove.yaml):
```json
{
  "gates": [
    { "name": "tests", "command": "npm test" },
    { "name": "typecheck", "command": "npm run typecheck" }
  ],
  "benchmarks": [
    {
      "name": "dogfood",
      "command": "lcm dogfood",
      "metrics": [
        {
          "name": "checks_passed",
          "extract": "grep -oP '\\d+(?=/39 passed)'",
          "direction": "higher_is_better",
          "tolerance": 0.02,
          "significance": 0.01
        }
      ]
    }
  ],
  "coverage_gate": {
    "command": "npx c8 report --reporter json",
    "threshold": 0.80,
    "changed_files": ["src/foo.ts"]
  },
  "regression_tolerance": 0.02,
  "significance_threshold": 0.01
}
```

Per-metric `tolerance` and `significance` override the global defaults.

### Output

Single JSON to stdout:
```json
{
  "verdict": "keep",
  "reason": "checks_passed improved 5.4%, no regressions",
  "gates": [
    { "name": "tests", "passed": true, "exit_code": 0, "duration_ms": 4200 }
  ],
  "metrics": {
    "checks_passed": {
      "baseline": 37,
      "candidate": 39,
      "delta_pct": 5.4,
      "direction": "higher_is_better"
    }
  },
  "improved": ["checks_passed"],
  "regressed": [],
  "verdict_logic": "no_regressions_and_at_least_one_improvement"
}
```

### Verdict values

| Verdict | Meaning |
|---|---|
| `gate_fail` | A hard gate (tests, typecheck, coverage) failed |
| `regress` | At least one metric regressed beyond tolerance |
| `neutral` | No regressions, but no metric improved beyond significance |
| `keep` | At least one metric improved, none regressed |

### Internal logic

1. Run gates sequentially, fast-fail on first failure -> `gate_fail`
2. Coverage gate: run after gates pass, before benchmarks
3. Run all benchmarks, extract all metrics via configured `extract` patterns
4. Set logic: any metric regresses > tolerance -> `regress`
5. No metric improves > significance -> `neutral`
6. At least one improves, none regress -> `keep`

### Baseline initialization mode

When baseline arg is `/dev/null` or missing, evaluate.sh runs gates and benchmarks but skips scoring. Outputs metrics-only JSON for the orchestrator to save as epoch-baseline.json and rolling-baseline.json.

## Loop Flow

```
SESSION START
  Orchestrator reads autoimprove.yaml
  Orchestrator generates evaluate-config.json (YAML -> JSON)
  Orchestrator runs: evaluate.sh config.json  ->  saves as epoch-baseline.json + rolling-baseline.json
  Orchestrator loads/creates state.json (trust tier, cooldowns, stagnation)
  Orchestrator loads experiments.tsv
  Crash recovery: scan orphaned worktrees, clean up, mark incomplete entries

LOOP (until budget exhausted or all themes stagnated):
  1. Check budget (experiment count) -> exhausted? -> STOP
  2. Pick theme (weighted random, respect cooldowns)
  3. Spawn experimenter:
     - Agent(isolation: "worktree", model: "sonnet")
     - Prompt: theme, scope, constraints (from trust tier), last 5 experiment summaries
     - NOT included: metric names, scoring logic, scores, evaluate-config.json
  4. Experimenter makes changes, commits, returns
  5. Orchestrator gets worktree path from Agent result
  6. Orchestrator diffs worktree -> changed_files list
  7. Orchestrator updates evaluate-config.json with changed_files for coverage gate
  8. Orchestrator runs: evaluate.sh config.json rolling-baseline.json
  9. Reads verdict JSON

  IF gate_fail -> delete worktree, log "fail"
  IF regress   -> delete worktree, log "regress"
  IF neutral   -> delete worktree, log "neutral", increment stagnation counter
  IF keep      -> rebase onto main (fast-forward), merge worktree,
                  update rolling-baseline.json, tag commit,
                  reset stagnation for theme, check trust tier escalation
                  (rebase failure -> treat as discard)

  10. Write experiments.tsv row
  11. Write experiments/<id>/context.json (full metric breakdown, SHA, prompt, model)
  12. Check epoch drift: compare rolling-baseline to epoch-baseline
      -> cumulative drift > 5%? -> HALT session
  13. Update cooldowns, check per-theme stagnation (5 consecutive non-improvements)
  14. All themes stagnated? -> EXIT EARLY
  15. -> LOOP

SESSION END
  Write summary to stdout (experiment counts, kept/discarded, drift, budget used)
```

## Updated autoimprove.yaml Schema (no weights)

Key change from DESIGN.md: `weight` fields removed. Per-metric `tolerance` and `significance` added.

```yaml
project:
  name: my-project
  path: .

budget:
  max_experiments_per_session: 20

gates:
  - name: tests
    command: npm test
  - name: typecheck
    command: npm run typecheck

benchmarks:
  - name: dogfood
    type: script
    command: lcm dogfood
    metrics:
      - name: checks_passed
        extract: "grep -oP '\\d+(?=/39 passed)'"
        direction: higher_is_better
        tolerance: 0.02
        significance: 0.01

themes:
  auto:
    strategy: weighted_random
    priorities:
      failing_tests: 5
      todo_comments: 3
      coverage_gaps: 2
      lint_warnings: 2
    cooldown_per_theme: 3

constraints:
  trust_ratchet:
    tier_0: { max_files: 3, max_lines: 150, mode: auto_merge }
    tier_1: { max_files: 6, max_lines: 300, mode: auto_merge, after_keeps: 5 }
    tier_2: { max_files: 10, max_lines: 500, mode: auto_merge, after_keeps: 15 }
    tier_3: { max_files: null, max_lines: null, mode: propose_only }
    regression_penalty: -1
  forbidden_paths:
    - autoimprove.yaml
    - test/benchmarks/**
    - test/fixtures/**
  test_modification: additive_only

safety:
  epoch_drift_threshold: 0.05
  coverage_gate:
    command: "npx c8 report --reporter json"  # user-supplied, must output per-file coverage
    threshold: 0.80
  regression_tolerance: 0.02
  significance_threshold: 0.01
  stagnation_window: 5
```

## Crash Recovery

On session start, before the loop:

1. Run `git worktree list --porcelain` to find all active worktrees
2. Filter for worktrees under `autoimprove/` paths
3. Cross-reference with experiments.tsv — worktrees with no entry or no verdict are orphans
4. Delete orphaned worktrees via `git worktree remove --force`
5. Clean up orphaned branches via `git branch -D` for matching `autoimprove/*` branches
6. Mark incomplete experiments.tsv entries as verdict `crash`
7. Recalculate experiment count from logged entries for budget tracking

## Test Project Fixture

Minimal Node.js project in `test-project/` with:
- A test suite (mix of passing tests and a few failing ones)
- A simple benchmark script outputting JSON metrics
- Known TODOs, lint warnings, and coverage gaps for the experimenter to find
- An `autoimprove.yaml` pre-configured for the fixture

Integration validation: run 3 experiments, verify at least 1 keep and 1 discard.

| Remote triggers / scheduled runs | v2 roadmap |
